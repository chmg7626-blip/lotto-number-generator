# 프로젝트 전용 스킬 예시

이 파일은 자리표시자다. 실제 스킬로 바꾸는 방법:

1. 폴더 이름과 frontmatter의 `name`을 스킬 이름으로 바꾼다 (소문자-하이픈).
2. `description`에 "언제 쓰는지"를 구체적으로 적는다 — 호출 여부는 이 문장으로 결정된다.
3. 본문에 절차를 단계별로 적는다. 작성 요령은 공통 스킬 writing-skills 참고.

프로젝트 전용 스킬로 좋은 예:
- 이 프로젝트의 배포 절차 (순서가 정해져 있고 빠뜨리면 사고 나는 것)
- 데이터 파일 갱신 절차
- 릴리스 노트 작성 형식
