# 로또 도구 UI의 신뢰·행동 패턴 — 외부 조사

- 조사일: 2026-07-15
- 출처 유형: 출처 있는 사실(공식·1차 출처·직접 확인) + 경쟁 제품 직접 확인
- 사실 근거 여부: 예 — 아래 근거에 한정
- 검증 상태: 후보 — 시각 방향과 우선순위는 사용자 확인 전 확정하지 않음
- 사용 근거: 시장·제품·접근성 패턴 비교

## 질문

로또 번호 생성기가 과장된 당첨 암시 없이도 사람의 관심을 끌고 전문적으로 보이려면, 첫 화면의 행동·최신 결과·고지·터치 제어를 어떻게 구성해야 하는가?

## 결론

- 첫 화면은 `생성 방식 → 게임 수 → 번호 뽑기`라는 한 가지 과업을 분명히 보여주는 편이 적합하다. 흥미 요소는 번호 공과 추첨 결과에 집중하고, 일반 제어부는 도구처럼 차분하게 유지한다.
- 최신 회차 번호는 장식용 hero가 아니라 출처가 있는 짧은 결과 브리핑으로 보여준다. 공식 결과가 우선한다는 고지는 결과·출처와 가까운 곳에서 읽을 수 있어야 한다.
- 모바일 반복 제어는 최소 44px, 가능하면 48px에 가까운 터치 영역을 확보한다. 현재 B안의 고정번호 44×44px은 최소 경계에 해당하며 더 작게 줄이지 않는다.
- 경쟁 생성기의 짧은 설명과 단일 `Generate` 행동은 참고할 수 있지만, 색·그래픽·문구를 복제하지 않고 이 프로젝트의 남색 잉크·금색 CTA·실제 로또 구간색 공으로 재구성한다.

## 근거

- 동행복권은 최신·이전 회차 확인과 직접 번호 입력에 의한 당첨 확인을 핵심 경로로 안내한다. 이 프로젝트에서도 최신 결과를 생성 행동과 가까운 신뢰 정보로 유지할 근거가 된다.
- Powerball 공식 번호 확인 화면은 입력 유효성뿐 아니라 공식 추첨 결과가 우선한다는 경계를 명시한다. 재미용 생성기에서도 결과·출처 고지를 약화하지 않아야 한다.
- Google Material 접근성 지침은 모바일 터치 대상을 48×48dp, 대상 사이 간격을 대체로 8dp 이상 권장한다.
- 미국 CMS Design System은 click/tap target을 44×44px 이상으로 요구하고 CTA 문구가 사용자가 하려는 행동과 일치해야 한다고 설명한다.
- National-Lottery.com의 번호 생성기 제품은 짧은 설명, 즉시 생성, 복수 세트 선택을 한 흐름에 배치한다. 이는 경쟁 제품 관찰이며 공식 복권 운영 기준으로 취급하지 않는다.

## 불확실성

- 공개 화면만으로 실제 전환율·재방문율·선호도를 판단할 수 없다.
- “사람들이 좋아할” 시각 방향은 사용자층과 기기별로 달라질 수 있다. 현재 제안은 전문성·즉시 사용성·기억성 사이의 후보 방향이다.
- 외부 사례의 색·레이아웃을 그대로 적용했을 때 이 프로젝트에서 같은 효과가 난다는 근거는 없다.

## 필요한 검증 (후보 의견일 때만)

- A안, B안, 제안한 hybrid polish를 390px·1280px에서 같은 상태로 비교하고 사용자가 최종 방향을 확인한다.
- 최소 3명에게 “첫 화면에서 무엇을 할 수 있는지”, “공식 서비스로 오해하는지”, “가장 먼저 누를 버튼이 무엇인지”를 묻는 짧은 task test를 한다.
- 구현 뒤 고정번호·결과 티켓·키보드·reduced-motion과 320px 경계를 다시 확인한다.
