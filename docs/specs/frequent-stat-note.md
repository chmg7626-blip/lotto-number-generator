## 목적

"역대 단골 번호" 모드로 뽑은 용지에서, 각 게임 아래에 그 게임에 섞인 단골 번호가 과거 추첨에서
몇 번 나왔는지 문장 한 줄로 보여준다. "역대 184번 나온 번호 34" 같은 읽는 재미를 주는 표시
전용 기능이다 — 예측이 아니라 과거 집계 사실만 말한다.

## 요구사항

1. **역대 단골 번호(frequent) 모드에서만** 표시한다 (2026-07-07 사용자 결정 — 순수 랜덤·행운수
   고정에는 붙이지 않는다). 표시 형태는 목업 B안: 게임 줄 아래 설명 한 줄.
2. 게임마다 그 게임 6개 중 **역대 출현 횟수 상위 2개**를 언급한다:
   "🔥 34는 역대 1,231번 추첨에서 184번 나온 단골 1위, 27은 181번(2위) 나왔어요" 형태.
   전체 회차 수는 게임당 첫 문장에만 넣는다(중복 제거).
3. 순위는 1~45 전체 기준이며, 출현 횟수 동률은 **공동 순위**(자기보다 횟수 많은 번호 수 + 1)로
   센다. 게임 내 언급 대상 선정에서 동률이면 번호가 작은 쪽을 먼저 언급한다(결정적 규칙).
4. 수치는 하드코딩하지 않고 로드된 draws 데이터에서 집계한다(주간 자동 갱신을 그대로 따라감).
   기존 calculateFrequencies를 재사용하고, 문구 생성은 RNG 없는 순수 함수로 만들어 결정적으로
   테스트한다.
5. 데이터가 없어 frequent 모드가 랜덤으로 폴백하는 경우에는 설명 줄을 표시하지 않는다.
6. 문구 어디에도 '당첨'·'보장'·'확률을 높'류 표현을 쓰지 않는다(사행성 규칙 — 기존 금지어
   테스트 패턴과 동일하게 테스트로 차단).
7. 설명 줄에 과거 집계 기준임이 드러나야 한다(예: "역대 1,231번 추첨에서" — 별도 면책 문구는
   기존 상단 면책 띠·모드 안내가 담당).

## 완료 조건

- [ ] frequent 모드로 뽑으면 각 게임 아래 설명 한 줄이 보이고, 게임당 상위 2개 번호의
      출현 횟수·순위가 실데이터 집계와 일치한다.
- [ ] random·fixed 모드로 뽑으면 설명 줄이 없다.
- [ ] 빈 draws(데이터 없음)로 frequent 모드를 뽑으면 설명 줄이 없다.
- [ ] 동률 처리: 같은 횟수인 번호들이 같은 순위로 계산된다 — 단위 테스트로 검증.
- [ ] 문구 생성 함수가 같은 입력에 항상 같은 문장을 돌려준다(순수 함수) — 단위 테스트.
- [ ] 문구 템플릿·생성 결과에 금지 표현('당첨'·'보장'·'확률을 높')이 없다 — 테스트로 검증.
- [ ] 복사(게임/전체) 텍스트는 지금처럼 번호만 담는다(설명 미포함).
- [ ] 기존 테스트 전부 통과 + lint·typecheck·build 통과.
