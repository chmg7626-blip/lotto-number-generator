- 확정 설계: docs/design/lotto-mvp.md

## 목적

클릭 한 번으로 로또 6/45 번호를 생성하고, 과거 당첨 데이터의 번호별 출현 빈도를 구경할 수 있는
정적 웹앱의 1차 MVP. 통계·가중 생성은 **예측이 아니라 재미·정보**다. React·하네스 워크플로우
학습과 공개 배포(GitHub Pages) 후 피드백 수집을 겸한다.

## 요구사항

1. **번호 생성 (순수 랜덤)**: 1~45 중 중복 없는 본번호 6개 + 나머지 39개 중 보너스 1개를 생성한다.
2. **빈도 통계 표시**: 과거 당첨번호 데이터(정적 JSON)에서 1~45 각 번호의 누적 출현 횟수를 계산해
   번호별 횟수와 CSS 막대로 보여준다. (1차는 이 한 가지 분석만)
3. **빈도가중 생성**: 사용자가 "자주 나온 번호 위주" 또는 "드물게 나온 번호 위주" 중 하나를 골라
   생성한다. 가중은 과거 출현 횟수 기반이며, **확률적 근거 없는 재미 요소**임을 UI에 명시한다.
4. **저장 (localStorage)**: 생성 결과를 저장하고 목록으로 조회한다. 개별 삭제와 전체 비우기를 지원한다.
5. **면책 배너**: 페이지 상단에 항상 보이는 면책 문구(재미용·당첨 보장 아님·통계는 예측 아님)를 둔다.
6. **데이터 형식 고정**: 과거 당첨번호 JSON 스키마를 고정해, 실데이터를 동일 형식으로 교체만 하면
   되게 한다. 실데이터 확보(출처·이용약관 확인)는 외부 조사로 사용자가 직접 수행한다(docs/research).

## 완료 조건

이 spec의 완료 조건은 설계와 독립적이다. 정상·오류·경계값까지 확인 가능한 형태로 둔다.

**번호 생성**

- [ ] 생성된 본번호는 항상 정확히 6개이고, 모두 1~45 범위이며, 서로 중복되지 않는다.
- [ ] 보너스 번호는 1개이고 1~45 범위이며 본번호 6개와 중복되지 않는다.
- [ ] 위 불변식(개수·범위·중복 없음)을 검증하는 단위 테스트가 있고 통과한다.

**빈도 통계**

- [ ] 주어진 당첨 데이터 JSON에 대해 1~45 각 번호의 누적 출현 횟수가 정확히 계산된다(테스트로 검증).
- [ ] 데이터가 비어 있어도(0건) 오류 없이 "데이터 없음" 상태를 표시한다(경계값).
- [ ] 각 번호의 횟수가 숫자와 CSS 막대(횟수에 비례한 길이)로 화면에 표시된다.

**빈도가중 생성**

- [ ] "자주 나온 번호 위주" 선택 시, 출현 횟수가 높은 번호가 더 자주 뽑히도록 가중된다.
- [ ] "드물게 나온 번호 위주" 선택 시, 출현 횟수가 낮은 번호가 더 자주 뽑히도록 가중된다.
- [ ] 가중 생성 결과도 본번호 6개 불변식(범위·중복 없음·개수)을 동일하게 만족한다(테스트로 검증).
- [ ] 데이터가 없으면 가중 생성은 순수 랜덤으로 폴백하고 "데이터 없음, 랜덤 동작" 상태를 표시한다(경계값, 설계에서 폴백으로 확정).
- [ ] 빈도가중 옵션 근처에 "확률적 근거 없는 재미 요소" 안내가 보인다.

**저장**

- [ ] 생성 결과를 저장하면 목록에 나타나고, 페이지를 새로고침해도 유지된다(localStorage).
- [ ] 저장 항목을 개별 삭제하면 그 항목만 목록과 저장소에서 사라진다.
- [ ] 전체 비우기를 하면 목록이 비고 저장소도 비워진다.
- [ ] 저장된 데이터가 없을 때 목록은 빈 상태를 오류 없이 표시한다(경계값).

**면책·표현**

- [ ] 페이지 상단에 면책 배너가 항상 보인다(재미용·당첨 보장 아님·통계는 예측 아님).
- [ ] 코드·UI·문구 어디에도 "당첨 확률을 높인다/당첨 보장" 류의 표현이 없다.

**품질 게이트**

- [ ] lint·typecheck·test·build 로컬 검사를 통과한다(완료 선언 전 verify).

## 범위 밖

- 연금복권 등 다른 복권 종류 (2차)
- 빈도 외 다양한 분석: 구간별·홀짝·연속번호·최근 추세 (2차)
- 특별 조합(생일·기념일 입력 기반) 생성 (2차)
- 로또 전문가 패러디 설명 버전 (2차)
- 차트 라이브러리 도입 (1차는 CSS 막대로 충분 — YAGNI)
- 가중치 강도 슬라이더 (1차는 자주/드물게 2지선다만)
- 서버·DB·로그인·계정 (저장은 localStorage만)
- 실데이터 자동 수집/크롤링 (데이터는 사용자가 외부에서 확인해 정적 JSON으로 동반)
