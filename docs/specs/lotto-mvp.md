## 목적

클릭 한 번으로 로또 6/45 번호를 생성하고, 과거 당첨 데이터의 번호별 출현 빈도를 구경할 수 있는
정적 **홈페이지형(히어로) 웹앱**의 1차 MVP. 통계·가중 생성은 **예측이 아니라 재미·정보**다.
React·하네스 워크플로우 학습과 공개 배포(GitHub Pages) 후 피드백 수집을 겸한다.

> 2026-06-29 범위 변경: `frontend-design` 인터뷰로 확정한 홈페이지 목업(docs/mockups/mockup-a-bright.html,
> 메모리 lotto-ui-redesign)을 반영한다. 저장 기능은 복사로 대체하고, 연속뽑기(5게임)·회차 당첨번호 띠·
> 당첨금액 표시를 추가한다. **추첨 애니메이션은 이번 범위 밖**(별도 spec)이고, **당첨번호·당첨금액은
> 실데이터 미확보 상태라 "샘플"로 표시**한다.

## 요구사항

1. **번호 생성 (순수 랜덤)**: 1~45 중 중복 없는 본번호 6개를 생성한다. **보너스 번호는 생성하지 않는다**
   (2026-06-29 결정). 과거 당첨 데이터(Draw)의 보너스는 실제 회차 정보라 별개로 유지하며 회차 당첨번호 띠에만 쓴다.
2. **연속뽑기 (1~5게임)**: `번호 뽑기` 시 ×1~×5 중 고른 만큼(기본 ×1, 최대 ×5) 게임을 생성해
   로또 용지 형태로 A~E 게임으로 보여준다. 각 게임은 본번호 6개이고, 모드(아래 3)는 모든 게임에 동일 적용한다.
3. **생성 모드 (3종)**: 사용자가 다음 중 하나를 골라 생성한다.
   - "순수 랜덤"
   - "역대 단골 번호" (과거 출현 횟수 높은 쪽 가중). 가중은 과거 출현 횟수 기반이며,
     **확률적 근거 없는 재미 요소**임을 UI에 명시한다.
   - "행운수 고정 (내 번호 직접 입력)": 1~45 번호판에서 0~5개를 직접 골라 고정하고, 나머지 자리는
     랜덤으로 채워 본번호 6개를 만든다. 0개 고정이면 순수 랜덤과 동일하다. 연속뽑기 ×N에서는
     **고정한 번호가 모든 게임에 공통으로** 들어가고, 나머지 자리만 게임별로 랜덤이다.
     (이전 "기념번호 조합"=드물게 가중 모드는 제거 — 이름·동작 불일치 해소, 2차로 미뤘던 입력 기반 조합을 1차로 당김.)
4. **빈도 통계 표시**: 과거 당첨번호 데이터(정적 JSON)에서 1~45 각 번호의 누적 출현 횟수를 계산해
   1~45 공 그리드로 출현 횟수를 보여준다. (1차는 이 한 가지 분석만)
5. **복사**: 각 게임을 개별 복사하고, 5게임 전체를 한 번에 복사할 수 있다(클립보드). 저장(localStorage)은
   1차에서 제외한다.
6. **회차 당첨번호 띠 (샘플)**: 상단에 한 회차의 당첨번호(본번호 6 + 보너스)를 띠로 표시한다.
   **실데이터 미확보 상태라 "샘플"임을 명시**한다.
7. **당첨금액 표시 (샘플)**: 하단에 등수별(1등 강조 + 2~5등) 당첨금액을 표시한다.
   **실데이터 미확보 상태라 "샘플"임을 명시**한다.
8. **면책 배너**: 페이지 상단에 항상 보이는 면책 문구(재미용·당첨 보장 아님·통계는 예측 아님)를 둔다.
9. **데이터 형식 고정**: 과거 당첨번호 JSON 스키마를 고정해, 실데이터를 동일 형식으로 교체만 하면
   되게 한다. 실데이터 확보(출처·이용약관 확인)는 외부 조사로 사용자가 직접 수행한다(docs/research).

## 완료 조건

이 spec의 완료 조건은 설계와 독립적이다. 정상·오류·경계값까지 확인 가능한 형태로 둔다.

**번호 생성·연속뽑기**

- [ ] 생성된 본번호는 게임마다 항상 정확히 6개이고, 모두 1~45 범위이며, 서로 중복되지 않는다.
- [ ] 생성 결과에 보너스 번호가 없다(본번호 6개만). 5게임 용지에도 본번호 6개만 표시한다.
- [ ] ×1~×5 중 고른 게임 수(기본 ×1, 최대 ×5)만큼 게임이 생성되어 A~E로 표시된다(경계값: ×1, ×5).
- [ ] 위 불변식(개수·범위·중복 없음)을 검증하는 단위 테스트가 있고 통과한다.

**생성 모드**

- [ ] "역대 단골 번호" 선택 시, 출현 횟수가 높은 번호가 더 자주 뽑히도록 가중된다.
- [ ] 가중 생성 결과도 본번호 6개 불변식(범위·중복 없음·개수)을 동일하게 만족한다(테스트로 검증).
- [ ] 데이터가 없으면 "역대 단골 번호" 생성은 순수 랜덤으로 폴백하고 "데이터 없음, 랜덤 동작" 상태를 표시한다(경계값, 설계에서 폴백으로 확정).
- [ ] "역대 단골 번호" 옵션 근처에 "확률적 근거 없는 재미 요소" 안내가 보인다.
- [ ] "행운수 고정" 선택 시에만 1~45 번호판이 보이고, 0~5개까지 고정 선택할 수 있다(6개째 선택은 차단 — 경계값).
- [ ] 고정한 번호가 결과 본번호 6개에 모두 포함되고, 나머지 자리는 고정 번호와 중복되지 않는 랜덤으로 채워진다(테스트로 검증).
- [ ] 0개 고정이면 순수 랜덤과 동일하게 본번호 6개가 생성된다(경계값).
- [ ] 연속뽑기 ×N에서 고정 번호는 모든 게임에 공통으로 들어가고, 나머지 자리만 게임별로 다르다.
- [ ] "행운수 고정" 생성 결과도 본번호 6개 불변식(범위·중복 없음·개수)을 동일하게 만족한다(테스트로 검증).

**빈도 통계**

- [ ] 주어진 당첨 데이터 JSON에 대해 1~45 각 번호의 누적 출현 횟수가 정확히 계산된다(테스트로 검증).
- [ ] 데이터가 비어 있어도(0건) 오류 없이 "데이터 없음" 상태를 표시한다(경계값).
- [ ] 1~45 각 번호의 출현 횟수가 공 그리드로 화면에 표시된다(빈도 높낮이가 시각적으로 구분된다).

**복사**

- [ ] 각 게임을 개별 복사할 수 있고, 복사된 텍스트에 그 게임의 본번호 6개가 들어 있다.
- [ ] 5게임 전체를 한 번에 복사할 수 있다.
- [ ] 복사 동작에 대한 피드백(복사됨 표시 등)이 보인다.

**회차 당첨번호·당첨금액 (샘플)**

- [ ] 상단에 한 회차 당첨번호(본번호 6 + 보너스)가 띠로 표시되고, "샘플"임이 명시된다.
- [ ] 하단에 등수별(1등 강조 + 2~5등) 당첨금액이 표시되고, "샘플"임이 명시된다.

**면책·표현**

- [ ] 페이지 상단에 면책 배너가 항상 보인다(재미용·당첨 보장 아님·통계는 예측 아님).
- [ ] 코드·UI·문구 어디에도 "당첨 확률을 높인다/당첨 보장" 류의 표현이 없다.

**품질 게이트**

- [ ] lint·typecheck·test·build 로컬 검사를 통과한다(완료 선언 전 verify).

## 범위 밖

- 연금복권 등 다른 복권 종류 (2차)
- 빈도 외 다양한 분석: 구간별·홀짝·연속번호·최근 추세 (2차)
- 생일·기념일에서 자동 파생하는 특별 조합 생성 (2차). 단, **번호를 직접 골라 고정하는 "행운수 고정"은 1차 포함**(요구사항 3).
- 로또 전문가 패러디 설명 버전 (2차)
- 차트 라이브러리 도입 (1차는 공 그리드로 충분 — YAGNI)
- 가중치 강도 슬라이더 (1차는 순수/자주/기념번호 3지선다만)
- 추첨 애니메이션 (번호뽑기→화면전환→머신 등장→하나씩 공개. 별도 spec — 메모리 lotto-draw-animation)
- 저장·계정 (1차는 복사로 대체. localStorage 저장/목록/삭제는 제외)
- 서버·DB·로그인
- 당첨번호·당첨금액 실데이터 (1차는 "샘플" 표시. 실데이터는 외부 조사로 출처·이용약관 확인 후 교체)
- 실데이터 자동 수집/크롤링 (데이터는 사용자가 외부에서 확인해 정적 JSON으로 동반)
