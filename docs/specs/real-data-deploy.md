## 목적

무작위 가짜 샘플 데이터를 동행복권 공식 출처 기반 실데이터(1회~최신 회차)로 교체하고,
GitHub Pages에 공개 배포해 실제 사용자 피드백을 받는다.

## 요구사항

- **당첨번호 실데이터**: 1회~최신 회차 전체. 기존 스키마 유지
  (`{ round, date?, numbers[6], bonus }` — src/data README의 `DrawsFile`).
- **당첨금 실데이터**: 최신 회차의 등수별(1~5등) 당첨금·당첨 게임 수로 `prize.sample.ts`를
  대체하고 UI의 "샘플" 표기를 제거한다. 당첨번호 데이터와 파일을 분리한다.
- **갱신 스크립트**: 동행복권 공식 데이터를 받아 위 스키마의 JSON으로 변환·갱신한다.
  구체적 취득 방법(엑셀 다운로드 URL/HTML 파싱/기타)은 설계에서 결정한다.
- **주간 자동 갱신**: GitHub Actions가 매주 추첨 이후 새 회차를 반영해 커밋하고, 그 결과가
  사이트에 자동 반영된다. 실패는 워크플로우 실패로 드러나야 한다.
- **출처·비공식 표기**: 앱 화면과 데이터 파일에 출처("당첨번호 출처: 동행복권 로또 6/45
  회차별 당첨번호"), 비공식·재배포 허가 아님, 권리자 요청 시 삭제 수용 문구를 넣는다.
- **GitHub Pages 배포**: main 기준 자동 배포. vite `base`를 `/lotto-number-generator/`로
  설정해 하위 경로에서 자산·오디오 로드가 깨지지 않게 한다.
- 기존 CI 게이트(lint·typecheck·test·build)를 통과한 것만 배포된다.

## 완료 조건

이 spec의 완료 조건은 설계와 독립적이다 — Claude·Codex 두 독립 설계안이 같은 기준으로 비교되도록,
요구사항을 정상·오류·경계값까지 확인 가능한 형태로 구체화한다. 검토자(Codex)와 사람이 이 기준으로 판정한다.

정상:

- [ ] 당첨번호 JSON이 1회~최신 회차를 포함하고, 스키마 검증 테스트를 통과한다
      (각 회차: 본번호 6개·중복 없음·1~45 범위·보너스 1~45, 회차 번호 누락 없이 연속).
- [ ] 빈도 통계·"역대 단골 번호" 생성이 실데이터 기준으로 동작하고, 가짜 샘플 import가
      코드에서 제거된다.
- [ ] 최신 회차 당첨번호 띠와 등수별 당첨금이 실데이터로 표시되고 "샘플" 표기가 사라진다.
- [ ] 갱신 스크립트를 로컬에서 실행하면 최신 회차까지 JSON이 갱신된다.
      멱등: 이미 최신이면 파일 변경이 없다.
- [ ] 주간 자동 갱신 워크플로우: 새 회차가 있으면 갱신 커밋이 생기고 사이트에 반영된다.
      새 회차가 없으면 커밋을 만들지 않는다.
- [ ] GitHub Pages URL(`https://chmg7626-blip.github.io/lotto-number-generator/`)에서
      사이트가 열리고, 하위 경로에서 JS·CSS·오디오·데이터 로드가 깨지지 않는다.
- [ ] 앱 화면에 출처 표기 + 비공식·삭제 수용 문구가 노출된다.

오류·경계:

- [ ] 데이터 취득 실패(네트워크 오류·응답 구조 변경) 시 스크립트가 0이 아닌 종료 코드로
      실패하고, 기존 JSON 파일을 손상시키지 않는다.
- [ ] 받아온 데이터가 스키마 검증에 실패하면(범위 밖 번호 등) 갱신을 반영하지 않는다.
- [ ] 당첨금 파일의 회차가 당첨번호 최신 회차와 일치하지 않으면 검증이 실패한다.
- [ ] 주간 자동 갱신 워크플로우는 생성 데이터 파일 외의 변경이 생기면 커밋하지 않고
      실패한다(diff allowlist).
- [ ] 기존 사행성 금지 표현 규칙 유지 — 금지어("당첨 보장"·"확률을 높") 테스트가 실데이터
      문구 추가 후에도 통과한다.
- [ ] CI(lint·typecheck·test·build)가 통과한다.

## 범위 밖

- 회차별 당첨금 조회 화면(당첨금은 최신 회차 표시용만).
- 연금복권 모드, 추가 분석(구간·홀짝·연속·추세).
- 동행복권 재배포 허가 문의(사용자가 위험 감수로 결정 — 추후 문의·회신 시 재검토).
- 커스텀 도메인, 방문 통계·애널리틱스.
