## 목적

추첨 연출(풀스크린 오버레이, 튜브 슛→대형 컷인→서스펜스→결과 컷)에 BGM과 효과음을 더해
몰입감을 소리로 강화한다. 번호 생성 로직·연출 흐름·결과는 바꾸지 않는다(소리는 연출에 얹기만 한다).

## 요구사항

1. **BGM**: 오버레이가 뜬 시점(믹싱)부터 연출 구간 동안 배경음악이 재생된다. 오버레이가 닫히면
   모든 소리가 멈춘다. (결과 컷 구간의 BGM 처리 — 지속/전환/페이드 — 는 설계에서 정한다)
2. **이벤트 효과음**: 연출의 주요 순간에 효과음이 붙는다. 최소 다음 세 순간은 소리로 구분돼야 한다:
   - 공 공개(튜브 슛~대형 컷인) — 공마다
   - 마지막 공 직전 서스펜스
   - 결과 컷 도달(팡파르류)
     (트레이 안착 등 추가 순간은 설계에서 정한다)
3. **기본 ON + 음소거 토글**: 소리는 기본 켜짐이다. 오버레이 안에서 언제든 음소거 토글에 접근할 수
   있고, 누르면 즉시 적용된다. 설정은 localStorage에 저장돼 다음 방문에도 유지된다.
   (2026-07-05 사용자 결정 — 기본 OFF·첫 뽑기 시 질문 아님)
4. **사용자 제스처 이후에만 재생**: 소리는 `번호 뽑기` 클릭(사용자 제스처) 이후에만 시작된다.
   페이지 로드·탐색만으로는 어떤 소리도 나지 않는다(브라우저 자동재생 정책과도 일치).
5. **건너뛰기와 소리 정리**: `건너뛰기`로 결과 컷에 즉시 도달하면 소리도 그 상태에 맞게 정리된다
   (진행 중이던 효과음이 어긋나게 잔류하지 않는다).
6. **소리 실패는 연출을 막지 않는다**: 음원 로드·재생이 실패해도 연출과 번호 공개는 소리 없이
   정상 진행된다.
7. **초기 로드 무부담**: 음원은 첫 화면 로드를 지연시키지 않는다(초기 번들에 음원을 포함하지 않고
   비동기로 준비 — 구체 방식은 설계에서 정한다).
8. **reduced-motion**: `prefers-reduced-motion: reduce`에서는 오버레이 자체가 생략되므로 소리도
   재생되지 않는다(기존 동작 유지).
9. **음원 출처·라이선스**: 외부 무료 음원을 쓸 수 있다(2026-07-05 사용자 결정). 단, 사용하는 모든
   음원은 상업적 사용·재배포가 허용되는 라이선스(CC0 등)여야 하고, 출처·라이선스를
   docs/research/에 기록한 뒤에만 도입한다. 저작자 표기 의무가 있는 라이선스면 UI 또는 저장소
   문서에 크레딧을 표기한다. 효과음을 Web Audio 합성으로 대체할지는 설계에서 정한다.
10. **테스트 가능성**: "어느 시점에 어떤 소리를 재생 요청하는가"는 재생 계층을 주입(모킹)해
    단위 테스트로 검증할 수 있어야 한다(실제 오디오 재생은 jsdom에서 검증 불가).

## 완료 조건

이 spec의 완료 조건은 설계와 독립적이다. 정상·오류·경계값까지 확인 가능한 형태로 둔다.

**재생 흐름**

- [ ] 소리 ON 상태에서 `번호 뽑기` 클릭 시 BGM이 시작되고, 공 공개(6회)·마지막 공 서스펜스·
      결과 컷 도달 시점에 각각 해당 효과음 재생이 요청된다(재생 계층 모킹 테스트로 검증).
- [ ] `확인`으로 오버레이가 닫히면 모든 소리가 멈춘다.
- [ ] 페이지 로드·모드 선택 등 뽑기 클릭 이전에는 어떤 소리도 재생 요청되지 않는다.

**음소거 토글**

- [ ] 연출 중 음소거 토글을 누르면 즉시 소리가 꺼지고, 다시 누르면 켜진다.
- [ ] 토글 상태가 localStorage에 저장되고, 새로고침 후에도 유지된다.
- [ ] 저장값이 없으면(첫 방문) 기본 ON이다(경계값).

**건너뛰기·오류**

- [ ] `건너뛰기` 시 결과 컷 상태에 맞게 소리가 정리되고, 어긋난 효과음이 잔류하지 않는다.
- [ ] 음원 로드 실패 상황에서도 연출·번호 공개·용지 반영이 정상 동작한다(소리만 없음).

**접근성·자산**

- [ ] `prefers-reduced-motion: reduce` 경로(오버레이 생략)에서는 소리 재생 요청이 없다.
- [ ] 사용한 모든 음원의 출처·라이선스가 docs/research/ 에 기록돼 있고, 표기 의무가 있으면
      크레딧이 표기돼 있다.
- [ ] 음원 파일이 초기 번들에 포함되지 않는다(빌드 산출물 확인).

**회귀·품질 게이트**

- [ ] 기존 연출·도메인 테스트가 그대로 통과한다(연출 흐름·번호 로직 무변경).
- [ ] lint·typecheck·test·build 로컬 검사를 통과한다(완료 선언 전 verify).

## 범위 밖

- 볼륨 슬라이더·세부 볼륨 설정 (음소거 토글만 — YAGNI)
- 사운드 테마 선택(음원 교체 옵션)
- 연출 밖 일반 UI 효과음(버튼 클릭음, 복사 완료음 등)
- 연속뽑기 B~E 게임의 개별 사운드(첫 게임만 연출이므로 해당 없음)
- 햅틱(진동) 피드백
- 연출 자체의 리듬·타이밍 변경(소리는 기존 타이밍에 맞춘다)
