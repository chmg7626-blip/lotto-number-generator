  도메인 로직은 건드리지 않지만 기존 뽑기 흐름 전체에 연출이 끼어들어 회귀 위험이 있고,

## 목적

`번호 뽑기` 클릭 시 결과가 용지에 즉시 채워지는 현재 동작을, **화면 전환 → 추첨기 등장 →
공이 하나씩 뽑혀 번호가 순차 공개**되는 본격 추첨 연출로 바꾼다. 실제 로또 추첨을 보는 듯한
재미·몰입감을 주기 위한 것이며, 번호 생성 로직·결과는 바꾸지 않는다(연출은 표시만 담당).

## 요구사항

1. **풀스크린 오버레이 추첨 무대**: `번호 뽑기` 클릭 시 화면 전체를 덮는 추첨 무대(어두운 배경 +
   큰 추첨기)가 떠오른다. 연출이 끝나고 닫히면 원래 화면의 용지에 결과가 채워져 있다.
   (2026-07-03 사용자 결정 — 페이지 씬 전환·히어로 내 확대가 아닌 오버레이)
2. **공 하나씩 공개**: 추첨기에서 공 6개가 하나씩 뽑혀 나오며 번호가 순차 공개된다.
   공개 순서는 뽑힌 순서(정렬 안 함)이고, 결과 컷과 용지에는 오름차순으로 정렬해 보여준다.
3. **번호는 연출 전에 확정**: 결과 번호는 연출 시작 전에 기존 도메인 로직으로 확정되고,
   연출은 그 번호를 공개만 한다. 건너뛰기 여부·연출 진행 정도와 무관하게 결과는 동일하다.
4. **연속뽑기 ×2~×5는 첫 게임만 풀 연출**: 게임 A만 공 6개 풀 연출로 보여주고, 닫힌 뒤
   B~E는 용지에 한 번에 채워져 있다. 연출 시간은 ×1과 동일하게 유지된다.
   (2026-07-03 사용자 결정 — 게임별 반복·전 게임 연출 아님)
5. **건너뛰기 버튼**: 오버레이에 `건너뛰기` 버튼을 두고, 누르면 연출을 즉시 끝내고 결과 컷을
   보여준다. (2026-07-03 사용자 결정)
6. **결과 컷 + 사용자 닫기**: 공 6개가 다 공개되면 뽑힌 번호 6개를 오름차순으로 정렬한 결과
   컷을 보여주고, `확인` 버튼으로 사용자가 닫는다(자동 닫기 아님). (2026-07-03 사용자 결정)
7. **모든 생성 모드 동일 적용**: 순수 랜덤·역대 단골·행운수 고정 모두 같은 연출을 쓴다.
   행운수 고정의 고정 번호도 다른 공과 똑같이 뽑혀 나온다(별도 취급 없음).
8. **연출 중 입력 차단**: 연출 중에는 뒤 화면을 조작할 수 없고, 뽑기 재클릭도 차단된다.
   오버레이 안에서 가능한 조작은 건너뛰기(연출 중)와 확인(결과 컷)뿐이다.
9. **접근성 — reduced-motion**: `prefers-reduced-motion: reduce` 환경에서는 연출 없이
   지금처럼 결과가 용지에 즉시 채워진다(오버레이 생략). 기존 "reduced-motion 시 애니메이션 끔"
   규칙과 일관.
10. **연출 길이**: ×1 기준 시작~결과 컷 도달까지 약 5~7초 목표. 반복 사용을 전제로 지루하지
    않은 길이를 유지한다. (당초 8~12초 → 2026-07-03 사용자 실사용 피드백 "공 뽑히는 속도가
    너무 느리다"로 단축)

## 완료 조건

이 spec의 완료 조건은 설계와 독립적이다. 정상·오류·경계값까지 확인 가능한 형태로 둔다.

**연출 흐름**

- [ ] `번호 뽑기` 클릭 시 풀스크린 오버레이 추첨 무대가 나타나고, 연출 중 뒤 화면(모드칩·뽑기
      버튼 등)은 조작할 수 없다.
- [ ] 공 6개가 하나씩 순차 공개되고, 공개 순서는 정렬되지 않은 뽑힌 순서다.
- [ ] 6개 공개 후 오름차순 정렬된 결과 컷이 보이고, `확인`을 누르면 오버레이가 닫히며 용지에
      같은 번호가 채워져 있다.
- [ ] 오버레이·결과 컷의 공은 기존 구간색(z1~z5) 규칙을 따른다.

**번호 불변**

- [ ] 연출로 공개되는 번호와 닫힌 뒤 용지의 번호가 일치한다(연출이 번호를 바꾸지 않는다 —
      테스트로 검증: 확정된 번호 입력 대비 공개 순서·정렬 결과가 순수 함수로 검증 가능해야 한다).
- [ ] 기존 본번호 불변식(6개·1~45·중복 없음)과 모드별 동작(가중·고정)은 기존 테스트가 그대로
      통과한다(도메인 로직 무변경).

**건너뛰기·연속뽑기**

- [ ] 연출 중 `건너뛰기`를 누르면 즉시 결과 컷으로 넘어가고, 번호는 건너뛰지 않았을 때와 동일하다.
- [ ] ×2~×5에서 게임 A만 연출되고, 닫힌 뒤 A~E 전 게임이 용지에 채워져 있다(경계값: ×1, ×5).
- [ ] 연출 중 `번호 뽑기`를 다시 누를 수 없다(차단 — 경계값).

**접근성·표현**

- [ ] `prefers-reduced-motion: reduce`에서는 오버레이 없이 결과가 즉시 용지에 채워진다.
- [ ] 연출·결과 컷의 문구 어디에도 "당첨 확률을 높인다/당첨 보장" 류 표현이 없다(기존 규칙 유지).

**품질 게이트**

- [ ] lint·typecheck·test·build 로컬 검사를 통과한다(완료 선언 전 verify).

## 범위 밖

- 효과음·배경음 (YAGNI — 필요해지면 별도 작업)
